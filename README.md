# 🟢 Ticket Maker CLI

**Ticket Maker** est un outil en ligne de commande (CLI) développé en **Rust** permettant de générer rapidement des tickets de gestion de projet au format **Markdown**. Il suit les conventions de structure des issues GitHub pour faciliter l'intégration dans un workflow de développement moderne.



## ✨ Fonctionnalités

- 📝 **Saisie interactive** : Création assistée de tickets via le terminal.
- 📂 **Batch Mode** : Créez plusieurs tickets en une seule session et sauvegardez-les tous d'un coup.
- 🏷️ **Typage Fort** : Gestion des types de tickets (Bug, Feature, Documentation, Refactor) via des Enums Rust.
- ✅ **Gestion de Tâches** : Ajout d'une To-Do list interactive (`- [ ]`) pour chaque ticket.
- 🕒 **Horodatage Automatique** : Utilisation de la crate `chrono` pour dater précisément la création.
- 💾 **Organisation Propre** : Sauvegarde automatique dans un dossier `/tickets` avec nommage en *kebab-case*.

## 🚀 Installation & Utilisation

### Prérequis
- [Rust & Cargo](https://rustup.rs/) (version stable)

### Lancement
1. Clonez le dépôt :
   ```bash
   git clone https://github.com/steevnPrm/ticket_maker.git
   cd ticket_maker
   ```
2. Lancez l'application :
   ```bash
   cargo run
   ```

## 🛠️ Structure du Projet

Le projet est découpé de manière modulaire :
- `src/main.rs` : Logique de l'interface utilisateur et boucle principale.
- `src/models.rs` : Définition des structures de données (`Ticket`), des types (`Enums`) et de la logique de rendu Markdown.
- `Cargo.toml` : Gestion des dépendances (ex: `chrono`).



## 📄 Exemple de Rendu
Chaque ticket généré ressemble à ceci :

```markdown
#NOM_DU_TICKET
Author: @steevn
Status: Open 🟢
Date: 2026-03-19 12:45

## 🚀 [FEAT] Nom du Ticket

### Description
Détails de la fonctionnalité à implémenter...

### 📋 Tasks
- [ ] Première tâche
- [ ] Deuxième tâche

---
*Generated by TicketMaker v1.0*
```
